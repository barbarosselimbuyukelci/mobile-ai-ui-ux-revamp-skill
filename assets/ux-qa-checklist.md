# Mobile UX QA Checklist

## Core Completeness

- [ ] Concept and production layers are both present
- [ ] Primary flow and fallback flows are documented
- [ ] All critical states are defined per screen

## Usability

- [ ] Primary task is obvious within seconds
- [ ] Error recovery is available at each failure point
- [ ] Interaction cost is minimized for frequent tasks

## Accessibility

- [ ] WCAG 2.2 AA contrast and semantics covered
- [ ] Keyboard and assistive navigation considered
- [ ] Touch targets and zoom resilience validated
- [ ] Reduced-motion behavior specified

## Explainable AI

- [ ] User can see what AI changed or generated
- [ ] User can see why the AI did it
- [ ] User can edit, override, or reject output

## Agentic UX

- [ ] Agent roles and handoffs are explicit
- [ ] Human takeover points are defined
- [ ] Context survives handoffs without loss

## Validation

- [ ] Heuristic review scored with evidence
- [ ] Usability test plan defined
- [ ] Success metrics and guardrails instrumented
