# Accessibility Baseline

## Standard

Target WCAG 2.2 AA for all production-ready outputs.

## Mobile Requirements

- Touch targets are reliably tappable.
- Focus order follows reading and action logic.
- Dynamic updates are announced when needed.
- Zoom and text scaling preserve function.
- Gestures have accessible alternatives when possible.

## Content Requirements

- Labels are explicit.
- Error messages provide recovery guidance.
- Instructions do not rely on color alone.
- Motion can be reduced without loss of meaning.

## Cognitive Accessibility

- Keep language direct and concrete.
- Keep screen goals singular where possible.
- Reduce hidden dependencies in multi-step tasks.
- Use progressive disclosure for advanced options.

## Release Checklist

- Keyboard and switch-access path passes critical flow
- Screen reader labels and status updates verified
- Contrast checks pass for all major states
- Motion preferences respected
- Error recovery path accessible and clear
